[Switch to architecture diagram]
"This application helps job seekers optimize their resumes by:
  1. Parsing PDF resumes and extracting skills and experience
  2. Calculating ATS (Applicant Tracking System) compatibility scores
  3. Matching resumes against job descriptions with percentage scores
  4. Generating personalized cover letters
  5. Comparing multiple job opportunities side-by-side

Architecture Explanation

"The system uses a serverless architecture with these AWS components:
  • Frontend: React application hosted on S3 with CloudFront CDN
  • API Layer: API Gateway routing requests to Lambda functions
  • Compute: Three Lambda functions (Parser, Calculator, Handler)
  • Storage: S3 for files, DynamoDB for analysis results
  • Security: CloudFront OAI for secure S3 access
The entire infrastructure is deployed using AWS SAM for Infrastructure as Code. Now let me show you the live application."

üìä PART 3: Results Explanation 3 minutes

ATS Score Section

"First, we see the ATS Compatibility Score - this one is 78 out of 100, which is in the GOOD range. The score is color-coded:
  • Green (80-100): Excellent
  • Yellow (60-79): Good
  • Orange (40-59): Fair
  • Red (below 40): Poor
Below the score, we provide specific feedback. This resume scores well because it has a dedicated Skills section, includes relevant keywords, and has good formatting."

Skills Match Section

"Next, we have the Skills Coverage visualization showing:
  • Matched skills: Technical skills from resume that appear in job
  • Unmatched skills: Skills in resume not relevant to this job
The percentage - 65% - means 65% of the candidate's skills match what the job requires."

Missing Keywords Section

"This section lists technical keywords from the job description NOT found in the resume. Notice these are all technical terms - Python, Docker, Kubernetes, CI/CD. Our algorithm uses pattern-based NLP to filter out soft skills, focusing only on concrete technical skills through:
  • CamelCase terms (e.g., JavaScript)
  • Hyphenated tech terms (e.g., CI/CD)
  • Version numbers (e.g., Python3)
  • Filtering out gerunds, abstract nouns, action verbs
This approach scales automatically - no hardcoded lists needed."

Cover Letter Section

"Finally, we automatically generate a personalized cover letter that:
  • Mentions specific matched skills
  • Acknowledges growth areas
  • Uses professional tone
  • Is completely editable
Users can edit directly, copy to clipboard, or download as PDF."

üí° PART 6: Code Highlights 1 minute

Pattern-Based Skill Detection

"Instead of hardcoded lists, we use linguistic patterns:
  • Words ending in -ing, -tion, -ment ‚Üí likely soft skills
  • CamelCase words ‚Üí likely technical (e.g., JavaScript)
  • This scales automatically to new technologies

TF-IDF Similarity

"We use Term Frequency-Inverse Document Frequency to calculate semantic similarity, not just keyword counting. This captures meaning, not exact matches."

Dynamic ATS Scoring

"Scores are context-aware:
  • Checks Skills section existence
  • Applies penalties based on skill count
  • Provides specific, actionable feedback
  • Varies by resume quality (not uniform)
